# Customer Support Domain Specification

## Scope
Assist users with legacy code modernization projects through conversational interface.

## Capabilities

### Translation Questions
- Explain why specific translation decisions were made
- Provide alternatives for ambiguous patterns
- Guide manual intervention when needed
- Show historical examples of similar translations

### Risk Explanations
- Break down risk scores into components
- Highlight specific code patterns causing risk
- Recommend mitigation strategies
- Provide industry best practices

### Progress Tracking
- Report current translation status
- Estimate completion time
- Identify blockers
- Suggest optimizations

### Configuration Guidance
- Help users set up language mappings
- Configure test coverage thresholds
- Customize risk evaluation rules
- Integrate with external systems

## User Interactions

### Query Types
- Status checks: "What's the progress on project X?"
- Explanations: "Why was this pattern flagged as risky?"
- Guidance: "How do I handle COBOL PERFORM statements?"
- Troubleshooting: "Translation failed for file Y, what should I do?"

### Response Patterns
- Provide direct answers first
- Offer additional context if helpful
- Link to relevant documentation
- Suggest next steps

## Integration Points
- Access to project database for status queries
- Translation history for examples
- Risk evaluation engine for explanations
- Documentation system for references
